## Обзор проекта

Основная цель этого ноутбука — преобразовать сырые, разнородные данные о пациентах в чистый, структурированный набор данных, готовый для построения прогностических моделей машинного обучения. В ходе анализа решаются следующие задачи:

*   Объединение данных из различных источников (справочники, измерения, события).
*   Очистка данных от выбросов и невалидных значений.
*   Анализ пропусков и их заполнение (импутация).
*   Создание новых признаков (например, `ИМТ`, `пульсовое_давление`).
*   Анализ распределений и корреляций.
*   Агрегация измерений на уровне пациентов для ML.
*   Формирование целевых переменных (бинарная, мультиклассовая, регрессионная) на основе количества клинически значимых событий.

## Данные

Анализ использует несколько CSV-файлов, которые должны находиться в папке `Downloads/`:

*   **`Идентификаторы Субъектов 10.04.2025.csv`**: Справочник субъектов РФ (`id субъекта`, `название субъекта`).
*   **`Идентификаторы МО 15.04.2025.csv`**: Справочник медицинских организаций (МО).
*   **`МО и Пациенты 10.04.2025.csv`**: Связка "пациент-программа", содержащая информацию о том, к какой группе наблюдения относится пациент (`группа наблюдения`: опыт, контроль, реальная практика).
*   **`Первичные данные от 10.04.2025.csv`**: Основной файл с измерениями (САД, ДАД, ЧП, рост, вес, дата/время измерения и т.д.). Это самый объемный файл.
*   **`Медикаментозная терапия от 10.04.2025.csv`**: Данные о назначенной терапии (МНН — международное непатентованное наименование).
*   **`Клинически значимые события от 15.04.2025.csv`**: Данные о произошедших КЗС у пациентов.

## Этапы работы

### Этап 1: Загрузка данных
Загрузка всех необходимых файлов с использованием `pandas`. Для большого файла `Первичные данные...` используется чтение по частям (chunks), чтобы избежать нехватки памяти.

### Этап 2: Проверка целостности
Проверка связей между таблицами и наличия дубликатов. На этом этапе происходит объединение таблицы `patient_program` со справочником субъектов для получения `id субъекта`. Также проверяется, все ли `id МО` из таблицы пациентов есть в справочнике МО.

### Этап 3: Очистка данных
Преобразование колонок с датами и временем в формат `datetime`. Преобразование числовых полей (САД, ДАД, ЧП, рост, масса) в числовой формат с обработкой ошибок. Расчет возраста пациента на момент измерения.

### Этап 4: Outlier Analysis
Выявление и анализ аномальных значений:
*   Физиологические границы для САД, ДАД, ЧП и возраста.
*   Проверка на будущие даты.
*   Анализ слишком частых измерений для одного пациента в день.

### Этап 5: Анализ по группам
Добавление информации о группе наблюдения к первичным данным. Анализ распределения пациентов по группам и базовой статистики измерений внутри каждой группы.

### Этап 6: Визуализация
Построение первичных графиков для понимания распределений основных показателей (САД, ДАД), их соотношения и активности измерений по часам.

### Этап 7: Анализ КЗС и терапии
Анализ таблицы с клинически значимыми событиями: топ-10 событий, статистика их количества на одного пациента. Анализ терапии: топ-10 назначаемых МНН.

### Этап 8: Сохранение и статистика
Удаление выбросов, выявленных на Этапе 4. Сохранение первых версий очищенных данных (`primary_clean.csv`, `patient_program.csv`, `kzs.csv`, `therapy.csv`) и вывод итоговой статистики по датасету.

### Этап 9-10: Детальная EDA и анализ по группам
Более глубокая визуализация и анализ, включая:
*   Детальные графики распределений с указанием средних и медиан.
*   Корреляционный анализ.
*   Сравнение поведения разных групп наблюдения (боксплоты, гистограммы, статистика).

### Этап 11: Анализ качества данных
Оценка качества собранных данных:
*   Процент пропусков в критических полях по группам.
*   Регулярность измерений (сколько месяцев пациент участвует в исследовании).
*   Анализ задержки передачи данных на сервер.

### Этап 12: Очистка дубликатов и анализ ночных измерений
*   **Удаление дубликатов:** Поиск и удаление записей с одинаковым `id пациента` и `время измерения`.
*   **Анализ ночных измерений:** Исследование доли измерений, приходящихся на ночное время (22:00 - 6:00). Анализ по часовым поясам и попытка коррекции времени, если данные хранятся в UTC.

### Этап 13: Импутация пропусков
Заполнение пропущенных значений:
*   Рост и вес заполняются медианными значениями для группы пациентов того же пола, возрастной группы и группы наблюдения.
*   Категориальные поля заболеваний заполняются значениями-заглушками ("Не указано", "Нет данных").
*   На основе заполненных данных создаются новые признаки: **ИМТ (индекс массы тела)** и **пульсовое давление**.
*   Проводится анализ и коррекция аномальных значений ИМТ.

### Этап 14: Финальные исправления и целевая переменная
Финальная подготовка данных для машинного обучения:
*   Исключение пациентов с аномальными значениями ИМТ (>50) или малым количеством измерений (<5).
*   Агрегация всех измерений по каждому пациенту в набор статистических признаков (среднее, медиана, стандартное отклонение, мин, макс, межквартильный размах) для САД, ДАД, ЧП и пульсового давления.
*   Добавление числа КЗС для каждого пациента.
*   **Создание целевых переменных:**
    *   `target_high_risk`: Бинарная переменная (1 - если число КЗС выше медианы).
    *   `target_risk_category`: Мультиклассовая переменная (Низкий, Средний, Высокий, Очень высокий риск).
    *   Регрессионная задача — предсказание абсолютного числа КЗС.

## Итоговые датасеты

В процессе работы создаются следующие ключевые файлы в папке `data/`:

1.  **`primary_clean.csv`**: Промежуточный файл с очищенными от выбросов первичными данными.
2.  **`primary_imputed_corrected.csv`**: Файл с первичными данными, где заполнены пропуски и созданы признаки ИМТ и пульсового давления.
3.  **`ml_dataset_final.csv`**: Датасет, агрегированный по пациентам, содержащий статистические признаки, но еще не готовый для обучения (могут быть пропуски).
4.  **`ml_dataset_ready.csv`**: **ИТОГОВЫЙ ДАТАСЕТ**. Готов для использования в моделях машинного обучения. Содержит финальные признаки и целевые переменные.

Все визуализации, созданные в процессе анализа, сохраняются в папку `data/photo/`.
